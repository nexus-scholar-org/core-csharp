Module documentation

Artifacts

Immutable artifact identity, provenance attachment, and path constraints for local bundle interoperability and reconstruction.

Scope and ownership

The Artifacts module is a domain-level primitive for declaring what an artifact is, how it is identified, and how its byte identity is verified. It is not a persistence module, and it does not perform filesystem IO.

  • Canonical artifact envelope fields: ref, logical path, kind, media type, byte size, raw digest, schema id/version.
  • Optional binding fields: source record digest, workflow node, provenance event id and digest, required-for field.
  • Stable logical path rules: no absolute or URI forms, no traversal segments, no backslashes, no empty segments, no leading/trailing slash.
  • Artifact raw-byte digest is computed directly from provided bytes using SHA-256 and compared exactly.

Gate and ADR status

Gate 6 uses ADR 0009 and is documented in GATE-06 plus GATE-06-EVIDENCE.

  • Manifest schema and scope: nexus.review-bundle.manifest, version 1.0.0, manifest digest scope bundle-manifest.
  • Artifact digest scope: raw-artifact-bytes via ContentDigest.Sha256(...).
  • Required logical-path bans include drive letters, URI forms, traversal, and empty or dot segments.
  • Known protocol/provenance digests are required for import-safe binding verification.

Core contracts

  • ArtifactDescriptor: canonical artifact metadata and path validation entry point.
  • ArtifactDescriptor.Create(...): constructor path and field validation, including fixed schema defaults only through overloads.
  • ArtifactDescriptor.ComputeRawByteDigest(...): SHA-256 of bytes.
  • ArtifactDescriptor.NormalizeLogicalPath(...) + TryValidateLogicalPath(...): strict logical path checks.
  • BundleArtifactEntry, ReviewBundleManifest, BundleProtocolBinding, BundleWorkflowBinding, BundleProvenanceBinding.
  • BundleVerifier enforces staged import validity, required-section presence, binding integrity, artifact tamper checks, and destructive-overwrite policy.

Validation and verification behavior

  • Path behavior: backslash, URI, absolute, traversal, dot/empty segment, leading/trailing slash paths are rejected.
  • Artifact field checks: blank field rejection, non-negative size, and SHA-256 digest format checks.
  • Verification flow: manifest identity + protocol binding + required schema checks + provenance checks + artifact bytes + overwrite checks.
  • Deterministic ordering: artifacts, schema refs, provenance bindings, membership, and candidate arrays are canonically ordered before hashing.
  • Error categories: includes invalid-artifact-path, duplicate-artifact-path, negative-artifact-size, invalid-artifact-digest, missing-artifact, checksum-mismatch, unsupported-required-schema, invalid-protocol-binding, invalid-provenance-binding, destructive-overwrite, and invalid-manifest-digest.

Fixtures and tests

  • Conformance positive fixtures in fixtures/conformance/artifacts and fixtures/conformance/bundles.
  • Conformance fixture assertions in BundleFixtureTests.cs.
  • Behavioral tests for strict path, digest scope, ordering stability, and verifier results in BundleTests.cs.
  • Boundary assertions in DependencyRulesTests.cs keep bundles/artifacts inward-only.

Related modules

  • Workflow (workflow declarations and artifact declarations carried into bundle manifest).
  • Provenance (event binding and event digest fields for bundle verification).
  • Shared (shared-identity membership payloads carried as reviewed candidates).
  • Bundles (assembly point for artifact manifest logic).

Known issues to track

  • No overwrite policy yet outside Gate 6: Gate 6 only rejects destructive overwrite by default and does not offer replacement policy variants.
  • No transport or container binding: manifest and artifact digests intentionally exclude archive and transport metadata by design.
  • No API/UI surfaces: import/export actions are fixture-backed and test-backed only; no CLI/API/plug-in execution surfaces are implemented in this gate.
