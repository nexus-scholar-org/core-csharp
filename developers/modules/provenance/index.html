Provenance

Append-only scientific event records that bind human/automation actors, activities, protocol/workflow context, input/output references, and canonical event digest material.

Scope and ownership

The Provenance module is the local evidence ledger for reconstructable events. It records immutable records only and intentionally excludes persistence, execution engines, and authority enforcement.

  • Records include agent, activity, subject, timestamp, optional protocol binding, optional workflow binding, ordered inputs/outputs, and event digest.
  • In-memory store is append-only with duplicate-event-id rejection and immutable snapshots.
  • Canonical checks reject non-canonical entity kinds for subject/input/output (cache/projection/wiki/local-path/generative items).
  • Event digest uses Kernel canonicalization and `DigestScope.ProvenanceEvent`.

Gate and ADR status

Primary governing docs: ADR 0008, GATE-05, and GATE-05-EVIDENCE.

  • Record schema: nexus.provenance-event with schema version 1.0.0.
  • Digest scope: provenance-event.
  • Stable error categories: duplicate-event-id, missing-actor, missing-required-input, missing-required-output, projection-not-canonical.
  • Non-claims remain explicit: no bundle parity, no persistence schema, no AI governance, no workflow execution, no provider/network behavior.

Core contracts

  • ResearchEvent: immutable event record with Agent, Activity, Subject, input/output refs, optional protocol/workflow bindings, and computed EventDigest.
  • ProvenanceActivity: declares whether actor/input/output are required for the activity.
  • ProvenanceAgent: actor identity and kind; canonical kinds include human/automation/plugin/system/import.
  • ProvenanceEntityRef: entity reference with canonical kind enforcement via blacklist of non-canonical projections.
  • ProvenanceProtocolBinding + ProvenanceWorkflowBinding: optional context references preserved by id and digest.
  • InMemoryProvenanceStore: append-only, deduplicating by event id.
  • ResearchEventFactory: validates required actor/input/output constraints and constructs event digest deterministically.

Validation and behavior details

  • Append-only semantics: store only appends and refuses duplicate event ids.
  • Digest computation: event-level canonical serialization includes event id, actor, activity, timestamp, subject, protocol/workflow bindings, inputs, outputs.
  • Entity kinds: subject/input/output kinds are validated as canonical through ProvenanceEntityRef.ValidateCanonicalKind.
  • Activity requirements: if activity requires actor/input/output, those fields must be present and input/output must include content digests.
  • Immutability: event records and store reads are cloned snapshots and exposed through immutable collections.

Related modules

  • Protocol: protocol identity and protocol-content digest are carried as optional provenance context.
  • Workflow: workflow identity and optional node-level references are recorded where applicable.
  • Bundles: uses event digests as optional provenance references during bundle verification.
  • Artifacts: provenance input/output digests can reference artifact identity.

Known issues to track

  • Scope is intentionally local. No persistence/API/UI/CLI import-export surface is implemented in this module.
  • Entity-kind projection model is strict. Non-canonical kinds are rejected even if operationally useful; this is by design for scientific reconstruction.
  • Store policy is minimal. There is no compaction, retention, archival rollover, or tamper-evidence envelope chain yet.
