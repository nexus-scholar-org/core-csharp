Architecture

Domain inward, applications outward

This page explains the dependency direction and authority boundaries behind the ecosystem.

Core boundary

Domain projects should not depend on EF Core, ASP.NET Core, UI frameworks, provider SDKs, storage SDKs, or concrete model clients. Infrastructure and applications depend inward.
